### 現状の全体像（このREADMEだけで把握できる要約）

このプロジェクトは、なでしこ3で作る「オンライン対戦のカードゲーム（舞星 / MAISTAR）」です。
サーバーはWebSocketでゲーム状態を一元管理し、クライアントは画面表示と操作入力を担当します。

#### 1. 役割分担

- server: ルーム管理、手番処理、ラウンド進行、得点計算、勝敗判定を担当
- client: 画面遷移（ホーム/待機/ゲーム）と、サーバーに送る操作イベントを担当
- data/effects: カード定義（JSON）とカード効果（なでしこ関数）の置き場

#### 2. 実行の流れ（予定）

- npm run dev で server と client を同時起動
- プレイヤーが部屋に参加
- 人数がそろったらゲーム開始
- サーバー主導でラウンド進行、各手番の行動を検証して反映
- 毎アクション後に最新状態を全員へ配信

#### 3. ゲーム進行仕様（確定している部分）

- 3ラウンド制
- 1R初期手札5枚、2Rは6枚、3Rは7枚
- 手番行動は「座敷 / 宣伝 / 紹介」から1つ
- 予約客はラウンド開始時に任意で1枚セット可能
- 予約客の使用は通常行動とは別枠（手番中の前後どちらでも可）
- 休業状態では座敷/予約客使用が不可（宣伝/紹介は可）、手番終了時に回復
- 終了条件A/B/Cあり（Aは即時、B/Cは「もう1巡」あり）
- Aボーナスは1・2Rで5点、3Rで10点
- 得点は「客収入合計 - 残り手札ペナルティ」、最低0点
- 残り手札ペナルティは累積（1枚=1, 2枚=3, 3枚=6...）
- 3R終了時の金子トークン合計で勝敗、同点は3Rの残り手札が少ない方が勝ち
- 対戦人数は3人以上対応
