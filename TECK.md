# なでしこの書き方メモまとめ？

## 前提知識

cnako=バックエンド向けの命令群<br>
wnako=フロントエンド向けの命令群

公式リポジトリにあるドキュメントのありか→node_modules\nadesiko3\doc<br>

## 外部プラグインの取り込み方

### <cnako向けプラグインの場合>

`npm install プラグイン名`でインストール後.nako3ファイルの先頭で<br>
`!『プラグイン名』を取り込む。`と記載する。

### <wnako向けプラグインの場合>

scriptタグのsrcにURLを貼る（プラグインのページに取り込み用のURLがある）

```
例：<script type="text/javascript" src="https://nadesi.com/v3/cdn.php?v=3.3.71&f=release/wnako3.js&run">
```

## 処理を書くときに使えそうな知識

**簡単な構文とかは公式みれば載ってるので見ましょう**

### <開発環境について>

- 「〇〇と表示」系の処理につく赤い波線のエラーは無視してOK<br>
  npm startで実行して出たエラーがちゃんとしたエラー

### <フロント関係>

- `<div id="status"></div>`という風にIDをあらかじめ振っておいて、なでしこ側で表示内容変更したいときは`「#status」へ「接続しました！」をテキスト設定。`とする。
- `ログDOM=「#logs」のDOM要素取得。`のようにID振ったやつをファイルの先頭で取得しまくる
- htmlは完全にタグとかだけにして、スクリプト部分はclient/〇〇.nako3に切り分ける

### <バック関係>

- BをWSサーバ全送信。
- 関数を作成した際の処理として`return`に相当するのが`戻す`
- 別ファイルに分けた機能の取り込み方

```
ファイルの先頭で!『ファイルの相対パス』を取り込む。
```

- JSONファイルの読み込み方

```
『ファイルの相対パス』を開いてテキストに代入。
```

### <両方使える処理>

- JSONとオブジェクト間の変換をする処理

```
AをJSONエンコードしてBに代入。｜AをJSONエンコードして表示。｜AをJSONエンコードしてWS送信。
AをJSONデコードしてBに代入。
```

- エラーだった時という処理

```
エラー監視
　　※ここに監視対象のプログラム
エラーならば
　　※ここにエラー時の処理
ここまで
```

- 辞書型変数の扱い

```
ネストしてなかったら@をつけて選択、ネストしてたら@いらない
AI:キー名が変数かどうかで使い分ける。ネストしたものはオブジェクトへのアクセスという扱いらしい
ネストなし：A@「url」をBに代入。
ネストあり：A[「name」][「id」]をBに代入。
```
